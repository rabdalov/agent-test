# Итерация 23 — Уведомление администратора о запросах от неавторизованных пользователей

## Цель

Реализовать механизм уведомления администратора при обращении неавторизованных пользователей к боту, с возможностью добавить или отклонить пользователя прямо из Telegram-интерфейса.

---

## Задачи

### 1. Модификация `app/handlers_karaoke.py` — обработка неавторизованных пользователей

- Обновить логику проверки `user_id` на вхождение в список разрешённых `TLG_ALLOWED_ID`.
- При обращении пользователя, не входящего в список разрешённых, запускать процесс уведомления администратора.

### 2. Реализация функции `_notify_admin_of_unauthorized_access`

- Функция принимает `user_id` и `user_name` неавторизованного пользователя.
- Отправляет сообщение администратору с `ADMIN_ID` с информацией о пользователе.
- Сообщение содержит inline-кнопки:
  - **«Добавить»** — добавляет `user_id` в список разрешённых.
  - **«Отклонить»** — добавляет `user_id` в список отклонённых.

### 3. Обновление метода `_reject_unauthorized`

- Перед отправкой отказа пользователю вызывать `_notify_admin_of_unauthorized_access`.
- Пользователю по-прежнему отправляется сообщение об отказе в доступе.

### 4. Структура хранения разрешённых и отклонённых пользователей

- Разработать структуру хранения для двух списков:
  - **Разрешённые** (`allowed`): пользователи, которым разрешён доступ.
  - **Отклонённые** (`rejected`): пользователи, которым отказано в доступе.
- Каждая запись содержит:
  - `user_id` — идентификатор пользователя Telegram.
  - `user_name` — имя пользователя (username или отображаемое имя).
- Хранение реализовать в файле (например, `users.json`) с персистентностью между перезапусками бота.
- Предусмотреть загрузку и сохранение списков при старте и изменении.

### 5. Игнорирование повторных запросов от отклонённых пользователей

- При обращении пользователя из списка отклонённых — игнорировать запрос (не отправлять уведомление администратору повторно, не отвечать пользователю).

---

## Проверка (критерии приёмки)

- При обращении пользователя, не входящего в список разрешённых (`TLG_ALLOWED_ID`), администратор получает уведомление в Telegram с:
  - `user_id` и `user_name` пользователя.
  - Кнопками **«Добавить»** и **«Отклонить»**.
- Нажатие **«Добавить»** добавляет пользователя в список разрешённых и сохраняет изменения.
- Нажатие **«Отклонить»** добавляет пользователя в список отклонённых и сохраняет изменения.
- Повторные запросы от пользователей из списка отклонённых игнорируются без уведомления администратора.
- Пользователь из списка разрешённых получает доступ к функциям бота в штатном режиме.

## Примечания

- Список `TLG_ALLOWED_ID` из конфигурации остаётся первичным источником разрешённых пользователей; динамически добавленные пользователи хранятся отдельно.
- Структура `users.json` должна поддерживать атомарную запись во избежание повреждения данных при конкурентном доступе.
- Логировать все события: уведомление администратора, добавление/отклонение пользователя, игнорирование повторного запроса.
