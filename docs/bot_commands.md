# Команды Telegram-бота

Бот принимает аудио- и видеофайлы (или HTTP-ссылки на них) и автоматически создаёт MP4-видео с синхронизированными субтитрами в стиле караоке. Управление процессом обработки осуществляется через команды ниже.

## Команды для вставки в BotFather

start - Приветственное сообщение. Объясняет, что бот делает и как его использовать.
search - Поиск трека в локальном хранилище или на Яндекс-музыке
continue - Продолжает обработку последнего трека с последней ошибки или остановки
step_download - Скачивание исходного аудио- или видеофайла
step_language - Запрос выбора языка у пользователя
step_separate - Разделение голоса и музыки
step_transcribe - Транскрибация вокальной дорожки
step_correct - Уточнение транскрипции через LLM
step_lyrics - Поиск и получение полного текста песни
step_align - Cопоставление текста песни с таймкодами слов.
step_ass - Генерация файла субтитров .ass
step_render - Сборка MP4-видео с субтитрами
step_send - Отправка готового видео пользователю

## Сценарии использования

### 1. Базовый сценарий

Отправьте боту аудио- или видеофайл в одном из поддерживаемых форматов (`MP3`, `FLAC`, `AVI`, `MP4`, `MKV`, длительность > 1 минуты) или прямую HTTP-ссылку на файл. Также поддерживаются ссылки с YouTube и Яндекс.Музыки.

Бот автоматически запустит полный пайпайн обработки:

```
DOWNLOAD → ASK_LANGUAGE → GET_LYRICS → SEPARATE → TRANSCRIBE → CORRECT_TRANSCRIPT → ALIGN → GENERATE_ASS → RENDER_VIDEO → SEND_VIDEO
```

По завершении каждого шага бот отправит статусное сообщение. После успешного прохождения всех шагов вы получите готовый MP4-файл с субтитрами в стиле караоке.

### 2. Продолжение после ошибки

Если один из шагов завершился с ошибкой:

1. Изучите сообщение об ошибке, которое прислал бот.
2. Устраните причину сбоя (например, проблему с внешним сервисом или входными данными).
3. Используйте `/continue` для повтора неудавшегося шага или одну из команд `/step_*` для запуска пайплайна с любого конкретного шага.

## Шаги пайплайна

| Шаг | Команда запуска | Описание |
|---|---|---|
| `DOWNLOAD` | *(запускается автоматически при получении файла/ссылки)* | Скачивание исходного аудио- или видеофайла из любого источника: Telegram-файл, HTTP URL, Яндекс.Музыка, YouTube, локальный файл. |
| `ASK_LANGUAGE` | `/step_language` | Запрос выбора языка песни у пользователя (Русский/English). Пайплайн приостанавливается до выбора. |
| `GET_LYRICS` | `/step_lyrics` | Получение текста песни: автоматический поиск через сервисы (Genius, Lyrica, lyrics-lib) или ручной ввод от пользователя. |
| `SEPARATE` | `/step_separate` | Разделение аудиодорожки на вокал и музыкальный фон с помощью **demucs**. |
| `TRANSCRIBE` | `/step_transcribe` | Транскрибация вокальной дорожки — распознавание слов с таймкодами через **speeches.ai**. |
| `CORRECT_TRANSCRIPT` | `/step_correct` | Уточнение транскрипции с использованием LLM на основе текста песни (повышает качество выравнивания). |
| `ALIGN` | `/step_align` | Сопоставление (выравнивание) эталонного текста песни с таймкодами транскрибированных слов. |
| `GENERATE_ASS` | `/step_ass` | Генерация файла субтитров в формате `.ass` с покадровой подсветкой слов. |
| `RENDER_VIDEO` | `/step_render` | Финальная сборка MP4-видео: наложение субтитров на исходное видео (или заставку) через **ffmpeg**. Создаётся видео с тремя аудиодорожками: Instrumental, Original, Instrumental+Voice. |
| `SEND_VIDEO` | `/step_send` | Отправка готового MP4-файла пользователю или формирование ссылки на скачивание. |
