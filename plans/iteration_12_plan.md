# Итерация 12 — Корректировка таймингов в шаге ALIGN

## Цель

Добавить в шаг `ALIGN` метод постобработки, который корректирует тайминги первых слов в строках, если их длительность превышает допустимый порог. Вместо длинной паузы перед первым словом строки вставляется специальный маркер `(проигрыш)`.

---

## Описание задачи

### Проблема

После выполнения сопоставления строк и слов с таймингами (шаг `ALIGN`) первое слово в строке может иметь аномально большую длительность. Это происходит, когда между концом предыдущей строки и началом первого слова текущей строки есть длинная инструментальная пауза (проигрыш). В таком случае тайминг первого слова «растягивается» назад, захватывая эту паузу.

### Решение

Добавить метод, который запускается **после** выполнения основного сопоставления строк и слов с таймингами. Метод проверяет длительность первых слов в каждой строке и при превышении порога `MAX_WORD_TIME` выполняет корректировку.

---

## Алгоритм корректировки

Для каждой строки (сегмента) проверяем первое слово:

1. Вычисляем длительность первого слова: `duration = end_time - start_time`
2. Если `duration > MAX_WORD_TIME`, выполняем корректировку:
   - Сохраняем `start_time_gap = start_time` (исходное начало первого слова)
   - Присваиваем первому слову новый `start_time = end_time - NORMAL_WORD_TIME`
   - **Перед** первым словом в строке вставляем новый элемент `(проигрыш)`:
     ```json
     {
       "word": "(проигрыш)",
       "start_time": <start_time_gap>,
       "end_time": <новый start_time первого слова>
     }
     ```
   - В разделе `segments` для соответствующей строки добавляем `(проигрыш)` в начало поля `text`

---

## Пример

### До корректировки

**Words:**
```json
[
  {
    "word": "myself",
    "start_time": 53.18,
    "end_time": 54.96
  },
  {
    "word": "Tell",
    "start_time": 54.96,
    "end_time": 60.6
  }
]
```

**Segments:**
```json
[
  {
    "text": "And, in the bad times, I fear myself",
    "start_time": 48.58,
    "end_time": 54.96
  },
  {
    "text": "Tell me something, boy",
    "start_time": 54.96,
    "end_time": 68.14
  }
]
```

Слово `"Tell"` имеет длительность `60.6 - 54.96 = 5.64` секунды, что превышает `MAX_WORD_TIME = 5`.

### После корректировки

- `start_time_gap = 54.96` (исходный `start_time` слова `"Tell"`)
- Новый `start_time` слова `"Tell"` = `60.6 - NORMAL_WORD_TIME`
- Перед `"Tell"` вставляется `(проигрыш)` с `start_time = 54.96`, `end_time = новый start_time "Tell"`

**Words:**
```json
[
  {
    "word": "myself",
    "start_time": 53.18,
    "end_time": 54.96
  },
  {
    "word": "(проигрыш)",
    "start_time": 54.96,
    "end_time": 59.4
  },
  {
    "word": "Tell",
    "start_time": 59.4,
    "end_time": 60.6
  }
]
```

**Segments:**
```json
[
  {
    "text": "And, in the bad times, I fear myself",
    "start_time": 48.58,
    "end_time": 54.96
  },
  {
    "text": "(проигрыш) Tell me something, boy",
    "start_time": 54.96,
    "end_time": 68.14
  }
]
```

## Параметры конфигурации

Добавить в `.env` (и `example.env`) следующие параметры:

| Параметр | Значение по умолчанию | Описание |
|---|---|---|
| `MAX_WORD_TIME` | `5` | Максимально допустимая длительность первого слова в строке (в секундах). При превышении выполняется корректировка. |
| `NORMAL_WORD_TIME` | *(задать)* | Нормальное время длительности первого слова в строке (в секундах). Используется для вычисления нового `start_time` первого слова. |

## Место реализации

- **Файл:** [`app/alignment_service.py`](../app/alignment_service.py)
- **Метод:** добавить новый приватный метод (например, `_fix_long_first_words`) в класс сервиса выравнивания
- **Вызов:** метод вызывается в конце основного метода выравнивания, после сопоставления строк и слов с таймингами
- **Конфигурация:** параметры `MAX_WORD_TIME` и `NORMAL_WORD_TIME` читаются из [`app/config.py`](../app/config.py)

---

## Критерии проверки

- При наличии слов с длительностью более `MAX_WORD_TIME` в начале строк происходит корректировка таймингов
- Перед первым словом строки добавляется элемент `(проигрыш)` с корректными `start_time` и `end_time`
- В разделе `segments` текст соответствующей строки начинается с `(проигрыш) `
- Исходный `start_time` первого слова сохраняется как `start_time` вставленного `(проигрыш)`
- Новый `start_time` первого слова вычисляется как `end_time - NORMAL_WORD_TIME`
- Слова с длительностью не более `MAX_WORD_TIME` не затрагиваются
