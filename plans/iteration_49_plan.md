# Итерация 49: Сравнение Debian 13 vs Ubuntu 24 для Docker-развёртывания

## Цель
Сравнить Debian 13 (trixie) и Ubuntu 24.04 LTS (Noble) как базовые образы для Docker-развёртывания с учётом **AMD GPU** (Rembrandt Radeon 680M).

> **Важно:** Пользователь имеет AMD GPU, не NVIDIA. ROCm (AMD) ≠ CUDA (NVIDIA).

## Сравниваемые образы

| Образ | База | Размер | Поддержка | Статус |
|-------|------|--------|-----------|--------|
| `debian:13-slim` | Debian Trixie | ~30 MB | Сообщество | Testing |
| `ubuntu:24.04` | Ubuntu Noble | ~25 MB | Canonical | LTS до 2029 |
| `python:3.12-slim` | Debian 12 Bookworm | ~50 MB | Docker Inc | Stable |
| `rocm/pytorch:rocm6.x_ubuntu24.04` | Ubuntu 24.04 | ~5 GB | AMD | Production |

## AMD GPU специфика (Rembrandt 680M)

- **Архитектура:** RDNA 2 (iGPU в Ryzen 6000)
- **Поддержка ROCm:** Частичная (начиная с ROCm 5.7)
- **Docker:** ROCm runtime требуется на хосте
- **PyTorch:** Доступен вариант `torch` с ROCm через pip/conda

**Вывод:** Для iGPU AMD проще и стабильнее использовать **CPU-only** вариант. ROCm в контейнерах требует сложной настройки и может не дать прироста.

## Анализ по критериям

### 1. GPU-поддержка для AMD (важно!)

#### ROCm на Debian 13 / Ubuntu 24.04

**Проблема:** AMD ROCm в Docker — сложнее чем NVIDIA CUDA:
- Требуется ROCm runtime на **хосте** (уже установлен `amd-smi`)
- Контейнер должен иметь доступ к `/dev/kfd` и `/dev/dri`
- Размер образа с ROCm: ~5 GB vs ~1 GB для CUDA

```dockerfile
# ROCm образ (огромный!)
FROM rocm/pytorch:rocm6.0_ubuntu24.04
```

#### Сравнение подходов

| Подход | Размер | Сложность | Производительность |
|--------|--------|-----------|-------------------|
| CPU-only | ~500 MB | Просто | ✅ Хорошая |
| ROCm (AMD) | ~5 GB | Сложно | ⚠️ Неизвестно для iGPU |
| CUDA (NVIDIA) | ~2 GB | Средне | ✅ Хорошая |

**Вывод для AMD Rembrandt:** CPU-only **оптимальнее** — меньше размер, проще деплой, стабильнее.

#### Ubuntu vs Debian для ROCm

| Критерий | Debian 13 | Ubuntu 24.04 |
|----------|-----------|--------------|
| ROCm поддержка | ⚠️ Ограниченная | ✅ Лучшая |
| AMD драйверы | Стабильные | Стабильные |
| ROCm в Docker | Оба сложны | Оба сложны |

**Документация AMD:** Ubuntu — рекомендуемая платформа для ROCm.

### 2. Версии пакетов

| Пакет | Debian 13 | Ubuntu 24.04 | Примечание |
|-------|-----------|--------------|------------|
| Python | 3.12 | 3.12 | Одинаково |
| FFmpeg | 6.1 | 6.1 | Одинаково |
| glibc | 2.38 | 2.39 | Ubuntu чуть новее |
| CUDA (opt) | 12.x | 12.x | Зависит от образа |

### 3. Размер образа

| Конфигурация | Размер | Примечание |
|--------------|--------|------------|
| `debian:13-slim` + UV + app | ~500 MB | Минимальный |
| `ubuntu:24.04` + UV + app | ~550 MB | Чуть больше |
| `rocm/pytorch:6.0` + app | ~5 GB | С ROCm (огромный!) |
| `python:3.12-slim` (текущий) | ~450 MB | Базовый |

**Вывод:** ROCm образ в 10 раз больше CPU-only — не оправдан для iGPU.

### 4. Стабильность и поддержка

| Критерий | Debian 13 | Ubuntu 24.04 |
|----------|-----------|--------------|
| Статус | Testing | LTS |
| Поддержка | ~2-3 года | 5+ лет |
| Security updates | Медленнее | Быстрее |
| Enterprise support | Нет | Canonical Pro |


### 6. GPU-варианты для AMD

#### CPU-only (рекомендуется)
```dockerfile
FROM ubuntu:24.04
# PyTorch CPU-only
RUN uv pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
```

#### ROCm (для экспериментов)
```dockerfile
FROM rocm/pytorch:rocm6.0_ubuntu24.04
# ROCm runtime требуется на хосте
# Доступ к /dev/kfd и /dev/dri
```

**Важно:** Для AMD iGPU (Rembrandt) CPU-only стабильнее и проще.

## 7. Оценка необходимости GPU для проекта караоке-бота

### 7.1 Ключевой вопрос: Нужен ли GPU вообще?

При анализе необходимости GPU необходимо учитывать:
- **Масштаб**: 1-2 пользователя максимум
- **Приоритет**: Максимальная стабильность (согласно [`docs/ADR-001. UV_vs_Conda.md`](docs/ADR-001.%20UV_vs_Conda.md))
- **Основная ML-задача**: Разделение аудио через `demucs`

### 7.2 Анализ задач проекта по использованию GPU

| Задача | Библиотека | GPU поддержка | CPU производительность | Вывод |
|--------|------------|---------------|------------------------|-------|
| **Разделение аудио** | `demucs` | ✅ CUDA, MPS | ✅ Отличная | CPU достаточно |
| **Анализ аудио** | `librosa` | ❌ Только CPU | ✅ Хорошая | GPU не нужен |
| **Сегментация** | `msaf` | ⚠️ Частично | ✅ Приемлемая | CPU достаточно |
| **Транскрибация** | `speeches.ai` (API) | N/A (внешний) | N/A | GPU не требуется |
| **Выравнивание текста** | Кастомный код | ❌ Нет | ✅ Мгновенно | GPU не нужен |
| **Генерация ASS** | Кастомный код | ❌ Нет | ✅ Мгновенно | GPU не нужен |
| **Рендеринг видео** | `ffmpeg` | ⚠️ NVENC/VA-API | ✅ Приемлемая | CPU достаточно |

### 7.3 Детальный анализ: demucs на CPU vs GPU

**Demucs** — единственная задача с потенциальной выгодой от GPU.

#### Производительность demucs (типичный трек 3-4 минуты)

| Конфигурация | Время обработки | Относительно |
|--------------|-----------------|--------------|
| CPU (8 ядер, modern) | 30-60 сек | baseline |
| CUDA (RTX 3060) | 10-20 сек | ~3x быстрее |
| ROCm (AMD iGPU) | 20-40 сек | ~1.5-2x быстрее |
| Apple Silicon (MPS) | 15-25 сек | ~2x быстрее |

**Важно для проекта**: При 1-2 пользователях разница между 30 и 10 секундами **некритична**. Пайплайн и так содержит другие долгие шаги (транскрибация через API, рендеринг видео).

### 7.4 Стоимость поддержки GPU

| Фактор | CPU-only | GPU (NVIDIA) | GPU (AMD ROCm) |
|--------|----------|--------------|----------------|
| **Сложность инфраструктуры** | ⭐⭐⭐ Простая | ⭐⭐ Средняя | ⭐ Сложная |
| **Стабильность** | ⭐⭐⭐ Высокая | ⭐⭐⭐ Высокая | ⭐⭐ Средняя |
| **Размер Docker образа** | ~500 MB | ~1.5 GB | ~5 GB |
| **Сборка образа** | ⭐⭐⭐ Быстрая | ⭐⭐ Средняя | ⭐ Медленная |
| **Воспроизводимость** | ⭐⭐⭐ Да | ⭐⭐⭐ Да | ⭐⭐ Частично |
| **Зависимости** | Минимум | CUDA runtime | ROCm runtime |
| **Отладка проблем** | ⭐⭐⭐ Просто | ⭐⭐ Средне | ⭐ Сложно |

### 7.5 Расчёт для 1-2 пользователей

**Сценарий**: Два пользователя отправляют треки одновременно.

**CPU-only подход**:
- Пайплайн выполняется последовательно
- Общее время: 2 × (download + demucs 45s + transcribe 30s + render 60s) ≈ 5 минут на трек
- Нагрузка на систему: предсказуемая

**GPU-подход**:
- Demucs быстрее на ~30 секунд
- Но добавляет сложность инфраструктуры
- ROCm для AMD требует настройки на хосте

**Вывод**: Экономия 30 секунд на треке **не оправдывает** усложнение инфраструктуры и потерю стабильности.

### 7.6 Соответствие принципам ADR-001

Согласно [`docs/ADR-001. UV_vs_Conda.md`](docs/ADR-001.%20UV_vs_Conda.md:23):

> **Использовать только UV, отказаться от MFA.**
> Приоритет проекта — **максимальная стабильность инфраструктуры**.

Тот же принцип применим к GPU:
- GPU (особенно ROCm) добавляет сложности
- Сложность = точки отказа
- Для 1-2 пользователей выигрыш в производительности минимален

## Решение

### Рекомендация по GPU: CPU-only (без GPU)

**Вывод по GPU для проекта караоке-бота:**

| Критерий | Оценка |
|----------|--------|
| **Необходимость** | ❌ GPU **не требуется** для 1-2 пользователей |
| **Практическая польза** | ⚠️ Минимальная (экономия ~30 сек на трек) |
| **Стабильность** | ✅ CPU-only максимально стабилен |
| **Простота деплоя** | ✅ CPU-only значительно проще |
| **Соответствие ADR-001** | ✅ Максимальная стабильность инфраструктуры |

**Окончательное решение**: Отказаться от GPU-ускорения в пользу максимальной стабильности.

---

### Рекомендация по базовому образу: Ubuntu 24.04 LTS (CPU-only)

**Обоснование для AMD GPU:**
1. **CPU-only оптимальнее:** ROCm образ в 10 раз больше, сложнее в настройке
2. **iGPU AMD:** Частичная поддержка ROCm, не гарантирует прироста
3. **LTS стабильность:** 5 лет поддержки vs Testing в Debian 13
4. **Будущее:** Если понадобится ROCm — проще мигрировать с Ubuntu

### Варианты реализации

#### Вариант A: CPU-only (рекомендуется, production)
```dockerfile
FROM ubuntu:24.04
# PyTorch CPU — стабильно, компактно, просто
RUN uv pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
```

#### Вариант B: ROCm-ready (для экспериментов, не production)
```dockerfile
FROM rocm/pytorch:rocm6.0_ubuntu24.04
# Требует ROCm runtime на хосте
# Только для тестирования, не для стабильного деплоя
```

> **Примечание**: Вариант B оставлен только для исследовательских целей. Для production используется только Вариант A.

## Задачи итерации

### 1. Создать Dockerfile.ubuntu24
```dockerfile
# syntax=docker/dockerfile:1
FROM ubuntu:24.04

ENV DEBIAN_FRONTEND=noninteractive \
    PYTHONUNBUFFERED=1

RUN apt-get update && apt-get install -y --no-install-recommends \
    python3.12 python3.12-venv python3-pip \
    ffmpeg libsndfile1 git build-essential \
    && rm -rf /var/lib/apt/lists/*

# Install UV
COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /usr/local/bin/

WORKDIR /app
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-dev

COPY app/ ./app/
RUN mkdir -p /tracks

ENV TRACKS_ROOT_DIR=/tracks
CMD ["uv", "run", "python", "-m", "app.main"]
```


## Итоговая рекомендация

> **Ubuntu 24.04 LTS с CPU-only, без GPU-ускорения**

### Ключевые выводы по GPU

1. **GPU не требуется** для проекта с 1-2 пользователями
2. **CPU-only оптимален** по соотношению стабильность/производительность
3. **ROCm для AMD iGPU** — сложный, нестабильный, не даёт значимого выигрыша
4. **Соответствие ADR-001**: максимальная стабильность инфраструктуры

### Итоговое решение

| Аспект | Решение |
|--------|---------|
| Базовый образ | `ubuntu:24.04` |
| GPU-ускорение | ❌ Отключено (CPU-only) |
| PyTorch | CPU-версия (`--index-url https://download.pytorch.org/whl/cpu`) |
| Размер образа | ~550 MB |
| Стабильность | Максимальная |

Ubuntu выбрана для совместимости с ROCm (если когда-нибудь понадобится экспериментировать), но production-деплой использует **только CPU**.

Debian 13 не даёт преимуществ для данного проекта, но добавляет риски Testing-ветки.
