- [x] **Итерация 12 — Корректировка таймингов в шаге ALIGN**
  - Добавить в шаг `ALIGN` метод, который запускается после выполнения сопоставления строк и слов с таймингами.
  - Метод должен проверить длительность первых слов в строках, и если длительность первого слова более "Max_word_time", то выполняется:
    1) Сохраняем "start_time_gap" ="start_time", "start_time" присваивается "end_time"-"normal_word_time"
    2) Перед первым словом в строке добавляется "(проигрыш)", у которого "start_time"="start_time_gap"
    3) В разделе Segments также вносим корректировку в соответствующую строку добавив перед первым словом "(проигрыш)"
  - Пример:
    Words:  [{
          "word": "myself",
          "start_time": 53.18,
          "end_time": 54.96
        },
    #начало добавленного блока
        {
          "word": "(проигрыш)",
          "start_time": 54.96,
          "end_time": 59.4
        },
    #конец добавленного блока
        {
          "word": "Tell",
          "start_time": 59.4,
          "end_time": 60.6
        },
    ],
    segments:[
        {
          "text": "And, in the bad times, I fear myself",
          "start_time": 48.58,
          "end_time": 54.96
        },
        {
          "text": "(проигрыш) Tell me something, boy",
          "start_time": 54.96,
          "end_time": 68.14
        },
    ]
  - Добавить параметры в .env:
    - "MAX_WORD_TIME" = 5 секунд  # время максимально допустимой длительности первого слова в строке
    - "NORMAL_WORD_TIME" # нормальное время длительности первого слова в строке
  - Проверка: при наличии слов с длительностью более MAX_WORD_TIME в начале строк, происходит корректировка таймингов с добавлением "(проигрыш)" участков.

- [x] **Итерация 17 — Корректировка транскрипции с использованием LLM (CORRECT_TRANSCRIPT)**
  - Добавить новый шаг CORRECT_TRANSCRIPT в пайплайн, который выполняется после TRANSCRIBE и перед ALIGN
  - Шаг использует LLM для корректировки транскрипции на основе загруженного текста песни
  - Цель: улучшить качество выравнивания текста путём предварительной корректировки распознанного текста
  - **План реализации:**
    1. Добавить параметры конфигурации для LLM (провайдер, API ключ, модель)
    2. Реализовать сервис LLM-корректировки транскрипции (CorrectTranscriptService)
    3. Интегрировать шаг CORRECT_TRANSCRIPT в KaraokePipeline
    4. Шаг принимает на вход:
       - Готовую транскрипцию в json(из шага TRANSCRIBE)
       - Загруженный текст песни (из шага GET_LYRICS)
    5. LLM анализирует транскрипцию и текст песни, возвращает скорректированную транскрипцию в json того же формата, что и исходная транскрипция
    6. Скорректированный текст сохраняется и передаётся на шаг ALIGN
  - **Проверка:** после выполнения шага CORRECT_TRANSCRIPT создаётся скорректированная транскрипция, которая затем используется в шаге ALIGN для более точного выравнивания.
