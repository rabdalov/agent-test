## Обзор

Документация по использованию Ubuntu 24.04 LTS (Noble) в качестве базового образа для Docker-развёртывания караоке-бота.

> **Важно:** Несмотря на наличие AMD GPU (Rembrandt Radeon 680M), проект использует **CPU-only** конфигурацию. GPU-ускорение не требуется для масштаба 1-2 пользователей и не оправдывает усложнения инфраструктуры (согласно [ADR-001](../ADR-001.%20UV_vs_Conda.md)).

## Необходимость GPU для проекта

**Вывод:** GPU не требуется для текущего функционала и масштаба проекта.

## Почему Ubuntu 24.04?

### Для AMD Rembrandt 680M iGPU

1. **CPU-only оптимальнее ROCm**
   - ROCm образ в 10 раз больше (~5 GB vs ~550 MB)
   - Сложная настройка runtime на хосте
   - Частичная поддержка iGPU не гарантирует прироста производительности

2. **LTS стабильность**
   - 5 лет поддержки (до 2029)
   - Быстрые security updates
   - Enterprise support от Canonical

3. **Совместимость с ROCm**
   - Если в будущем понадобится GPU — проще мигрировать
   - Ubuntu — рекомендуемая платформа AMD

4. **Debian 13 (trixie)** — Testing ветка
   - Риски нестабильности
   - Нет гарантий долгосрочной поддержки
   - Меньше документации для ROCm

### Для данного проекта (1-2 пользователя)

| Критерий | Требование | Решение |
|----------|------------|---------|
| Производительность | Приемлемая | CPU достаточно |
| Стабильность | Максимальная | CPU-only |
| Простота деплоя | Высокая | Без GPU runtime |
| Размер образа | Компактный | ~550 MB |

## Быстрый старт

### Сборка образа

```bash
# CPU-only (рекомендуется)
docker build -f Dockerfile.ubuntu24 -t karaoke-bot:ubuntu24 .

# ROCm (экспериментально)
docker build -f Dockerfile.ubuntu24-rocm -t karaoke-bot:rocm .
```

### Запуск через Docker Compose

```bash
# CPU-only
docker-compose -f docker-compose.ubuntu24.yaml up -d

# Просмотр логов
docker-compose -f docker-compose.ubuntu24.yaml logs -f

# Остановка
docker-compose -f docker-compose.ubuntu24.yaml down
```

### Запуск напрямую (docker run)

```bash
docker run -d \
  --name karaoke-bot \
  --env-file .env \
  -v $(pwd)/tracks:/tracks \
  -v $(pwd)/.env:/app/.env:ro \
  karaoke-bot:ubuntu24
```

## Конфигурация GPU

### NVIDIA GPU (опционально)

Раскомментируйте в `docker-compose.ubuntu24.yaml`:

```yaml
deploy:
  resources:
    reservations:
      devices:
        - driver: nvidia
          count: 1
          capabilities: [gpu]
```

Требуется:
- NVIDIA Container Toolkit на хосте
- Драйверы NVIDIA

### AMD ROCm (экспериментально)

Для запуска с AMD GPU:

```bash
# Требуется ROCm runtime на хосте (amd-smi)
# Доступ к /dev/kfd и /dev/dri

docker run -d \
  --name karaoke-bot-rocm \
  --device=/dev/kfd \
  --device=/dev/dri \
  --group-add video \
  --security-opt seccomp=unconfined \
  --env-file .env \
  -v $(pwd)/tracks:/tracks \
  karaoke-bot:rocm
```

**Важно:** Для AMD iGPU (Rembrandt) CPU-only вариант стабильнее и проще!

### FFmpeg 8.0

В итерации 53 принято решение использовать FFmpeg 8.0 через статическую сборку:

**Преимущества FFmpeg 8.0:**
- До 15% быстрее рендеринг при `tune=stillimage`
- Улучшенное качество AAC кодирования (256-320 kbps)
- Исправления многодорожечного аудио (метаданные 4+ дорожек)
- Более качественный рендеринг ASS-субтитров (karaoke-эффекты)

**Установка:** Статическая сборка от johnvansickle.com
```dockerfile
RUN wget -q https://johnvansickle.com/ffmpeg/releases/ffmpeg-8.0-amd64-static.tar.xz \
    && tar -xf ffmpeg-8.0-amd64-static.tar.xz \
    && mv ffmpeg-8.0-amd64-static/ffmpeg /usr/local/bin/ffmpeg \
    && mv ffmpeg-8.0-amd64-static/ffprobe /usr/local/bin/ffprobe
```

**Почему не Ubuntu 25.04?**
- Ubuntu 25.04 содержит FFmpeg 7.1.x, а не 8.0
- Всего 9 месяцев поддержки (interim release)
- Статическая сборка на Ubuntu 24.04 LTS — оптимальный баланс версии и стабильности

## Рекомендации

### Production (единственный рекомендуемый вариант)

- **Образ:** `Dockerfile.ubuntu24` (CPU-only)
- **Причина:**
  - Максимальная стабильность (соответствие ADR-001)
  - Компактный размер (~550 MB)
  - Простота деплоя без GPU runtime
  - Достаточная производительность для 1-2 пользователей
- **GPU:** Не требуется и не рекомендуется

### Эксперименты с GPU (не для production)

- **Образ:** `Dockerfile.ubuntu24-rocm`
- **Причина:** Исследовательские цели, тестирование ROCm
- **Важно:** Только при наличии дискретной AMD GPU и опыта с ROCm

### Будущее развитие

Если в проекте появятся задачи, критичные к GPU:

1. **Оценить реальный прирост**: Замерить экономию времени vs усложнение
2. **Для 1-2 пользователей**: Скорее всего, CPU останется оптимальным
3. **При масштабировании**: Рассмотреть NVIDIA (CUDA) — стабильнее ROCm
4. **Сохранить CPU fallback**: Всегда иметь работающий CPU-only вариант

## Итоговое решение

> **Ubuntu 24.04 LTS + CPU-only = оптимально для данного проекта**

### Почему без GPU?

1. **Масштаб 1-2 пользователей** — CPU справляется без задержек
2. **Прирост от GPU минимален** — экономия ~30 сек на фоне 3-5 мин пайплайна
3. **Соответствие ADR-001** — максимальная стабильность инфраструктуры
4. **Сложность ROCm** — не оправдана для iGPU AMD

### Что выбрано

| Компонент | Решение | Обоснование |
|-----------|---------|-------------|
| Базовый образ | `ubuntu:24.04` | LTS до 2029, стабильность |
| FFmpeg | **8.0 (static build)** | Производительность, исправления |
| GPU | ❌ Отключён | Не требуется для задач |
| PyTorch | CPU-only | Стабильнее, компактнее |
| Размер образа | ~600 MB | (+50 MB за FFmpeg 8.0) |

### Почему FFmpeg 8.0?

Согласно [ADR-003](../ADR-003.%20FFmpeg_8.0_Priority.md):
- До 15% быстрее рендеринг
- Улучшенное качество AAC
- Исправления многодорожечного аудио
- Ubuntu 25.04 не подходит (содержит FFmpeg 7.1.x)

### Если понадобится GPU в будущем

Ubuntu 24.04 выбрана как платформа с лучшей совместимостью с ROCm — миграция возможна при реальной необходимости и доказанной выгоде.
