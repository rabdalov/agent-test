# Анализ класса ChorusDetector

## Обзор

Класс `ChorusDetector` реализует три различных подхода для обнаружения припевов в аудиофайлах: MSAF (Music Structure Analysis Framework), Librosa-based feature analysis и гибридный подход, объединяющий оба метода.

Класс принимает путь к аудиофайлу и возвращает подробную информацию о обнаруженных сегментах, включая тайминги, тип сегмента (припев/не припев), используемый бэкенд и бэкенд-специфичные метрики. Каждый бэкенд использует различные алгоритмы и признаки для идентификации сегментов припева, а гибридный подход пытается использовать преимущества обоих отдельных методов.

Реализация включает всестороннюю обработку ошибок, фильтрацию по длительности и подробное логирование на протяжении всего процесса обработки. Класс спроектирован как настраиваемый с различными бэкендами и ограничениями по длительности, чтобы соответствовать различным потребностям.

## Структура класса и инициализация

Класс `ChorusDetector` инициализируется с тремя параметрами:

1. `backend`: Строка, определяющая алгоритм обнаружения ("msaf", "librosa" или "hybrid"; по умолчанию "hybrid")
2. `min_duration_sec`: Минимальная длительность сегмента (по умолчанию 15.0 секунд)
3. `max_duration_sec`: Максимальная длительность сегмента (по умолчанию 60.0 секунд)

## Публичные методы

### `detect_with_info(self, audio_file: str) -> list[SegmentInfo]`
- **Назначение**: Главная точка входа, обнаруживающая сегменты припевов с подробной информацией
- **Вход**: Путь к аудиофайлу (строка)
- **Выход**: Список объектов `SegmentInfo`, содержащих:
  - Времена начала/окончания в секундах
  - Тип сегмента ("chorus" или "non-chorus")
  - Используемый бэкенд ("msaf", "librosa" или "hybrid")
  - Словарь метрик, специфичных для бэкенда

## Приватные методы бэкендов

### `_detect_msaf_with_info(self, audio_file: str) -> list[SegmentInfo]`
- **Назначение**: Реализует обнаружение припевов с использованием библиотеки MSAF с кластеризацией спектра
- **Обработка**:
  1. Загружает аудиофайл через MSAF
  2. Выполняет сегментацию структуры с использованием границ "sf" и меток "scluster"
  3. Строит сегменты из границ и меток
  4. Определяет сегменты припевов на основе наиболее частой/повторяющейся метки
  5. Фильтрует по ограничениям длительности
- **Доступные параметры MSAF для анализа**:
  - `boundaries`: Временные границы сегментов, определенные алгоритмом
   - `labels`: Метки сегментов, присвоенные кластеризацией; каждая метка представляет собой целочисленный идентификатор сегмента, присваиваемый на основе спектрального сходства между сегментами; сегменты с одинаковыми метками имеют схожие спектральные характеристики; припев обычно получает метку, которая встречается наиболее часто в треке (наиболее повторяющийся сегмент); алгоритм использует спектральную кластеризацию (spectral clustering) для группировки сегментов с похожими акустическими характеристиками; метки присваиваются на основе схожести спектральных признаков (таких как MFCC, спектральная центроида, спектральная ширина и т.д.)
  - `label_count`: Количество вхождений каждой метки (показывает повторяемость сегментов)
  - `label_duration`: Общая длительность сегментов с одинаковой меткой (помогает определить доминирующие части трека)
  - `segment_timings`: Временные диапазоны каждого сегмента (start_time, end_time)
- **Метрики**: Содержит ID метки, количество и длительность

### `_detect_librosa_with_info(self, audio_file: str) -> list[SegmentInfo]`
- **Назначение**: Реализует обнаружение припевов с использованием признаков Librosa
- **Обработка**:
  1. Загружает аудио с помощью Librosa
  2. Вычисляет хромаграмму (chroma_cqt)
  3. Создает матрицу самоподобия
  4. Определяет границы сегментов через агломеративную кластеризацию
  5. Ранжирует сегменты по комбинации признаков:
     - повторяемость (оценка самоподобия)
     - энергия гармоники (HPSS)
     - стабильность темпа (tempogram)
  6. Выбирает сегменты с наивысшим общим рейтингом как припевы
- **Доступные параметры Librosa для анализа**:
  - `chroma_features`: Хромаграмма (12xT матрица), представляющая распределение энергии по 12 полутонам
  - `self_similarity_matrix`: Квадратная матрица (T×T) с коэффициентами сходства между временными фреймами
  - `segment_boundaries`: Временные границы сегментов, определенные кластеризацией
  - `similarity_scores`: Оценки самоподобия для каждого сегмента (на основе среднего значения в блоке матрицы)
  - `hpss_harmonic_energy`: Энергия гармонической составляющей для каждого сегмента (после разделения HPSS)
  - `tempogram_stability`: Ритмическая стабильность для каждого сегмента (на основе анализа темпа)
  - `total_segment_scores`: Комбинированные оценки для каждого сегмента (similarity + hpss + tempo)
  - `threshold`: Пороговое значение (медиана оценок), используемое для классификации сегментов как припев/не-припев
- **Метрики**: Содержит sim_score, hpss_score, tempo_score, total_score и порог

### `_detect_hybrid_with_info(self, audio_file: str) -> list[SegmentInfo]`
- **Назначение**: Объединяет результаты бэкендов MSAF и Librosa
- **Обработка**:
  1. Запускает оба детектора
  2. Если оба успешны, объединяет сегменты припевов с пересечением >30%
  3. Если только один успешен, использует его результат
  4. Объединяет сегменты не-припевов из обоих бэкендов
- **Метрики**: Содержит поле "confirmed_by", указывающее источник ("librosa", "msaf", "librosa_fallback")

## Вспомогательные методы для вычисления признаков

### `_compute_chroma(self, y: np.ndarray, sr: int) -> np.ndarray`
- Вычисляет хромаграмму Constant-Q Transform (матрица 12xT)

### `_compute_self_similarity(self, chroma: np.ndarray) -> np.ndarray`
- Создает матрицу самоподобия с использованием косинусного сходства между фреймами

### `_compute_boundaries(self, y: np.ndarray, sr: int, chroma: np.ndarray, duration: float) -> list[float]`
- Определяет границы сегментов с использованием агломеративной кластеризации на признаках хромы

### `_hpss_energy(self, y: np.ndarray, sr: int) -> np.ndarray`
- Вычисляет энергию гармонической составляющей с использованием HPSS и нормализации RMS

### `_compute_tempogram_stability(self, y: np.ndarray, sr: int, boundaries_sec: list[float]) -> dict[int, float]`
- Вычисляет ритмическую стабильность для каждого сегмента с использованием анализа темпа

## Вспомогательные методы для обработки сегментов

### `_build_segments_from_boundaries(self, boundaries: list[float], labels: list[int]) -> list[_Segment]`
- Преобразует точки границ и метки в кортежи (start, end, label)

### `_pick_chorus_segments(self, segments: list[_Segment]) -> list[tuple[float, float]]`
- Выбирает сегменты с наиболее часто встречающейся меткой (считаются припевами)

### `_filter_by_duration(self, segments: list[tuple[float, float]]) -> list[tuple[float, float]]`
- Фильтрует сегменты по ограничениям минимальной/максимальной длительности

### `_merge_segments_with_source(self, msaf_segs: list[tuple[float, float]], librosa_segs: list[tuple[float, float]]) -> list[tuple[tuple[float, float], str]]`
- Объединяет сегменты из обоих бэкендов с атрибуцией источника

## Последовательность обработки

1. **Инициализация**: Установка типа бэкенда и ограничений длительности
2. **Обнаружение**: Вызов `detect_with_info()` направляет к соответствующему методу бэкенда
3. **Обработка бэкенда**:
   - **MSAF**: Использует кластеризацию спектра для сегментации структуры
   - **Librosa**: Использует признаки хромы, самоподобия и несколько метрик оценки
   - **Hybrid**: Объединяет результаты из обоих бэкендов с проверкой пересечений
4. **Фильтрация**: Применение ограничений длительности к сегментам
5. **Вывод**: Возврат списка объектов `SegmentInfo` с подробными метаданными

## Классы данных

### `SegmentInfo`
Содержит подробную информацию о каждом обнаруженном сегменте:
- Времена начала/окончания
- Тип сегмента ("chorus"/"non-chorus")
- Используемый бэкенд
- Метрики, специфичные для бэкенда

### `VolumeSegment`
Представляет временные сегменты с уровнями громкости вокала:
- Времена начала/окончания
- Уровень громкости (0.0-1.0)
- Информация о типе сегмента и бэкенде

Класс поддерживает три различных подхода к обнаружению припевов, позволяя пользователям выбирать в зависимости от своих потребностей и вычислительных ресурсов. Гибридный подход пытается объединить преимущества обоих методов MSAF и Librosa для улучшения точности.

## Параметры для исследования и анализа

### Для MSAF-бэкенда:
- Наиболее полезные параметры для анализа:
  - `label_count`: Показывает, какие сегменты повторяются чаще всего (потенциальные припевы)
  - `label_duration`: Помогает определить доминирующие по длительности части трека
  - `boundaries`: Показывает, как алгоритм делит трек на структурные единицы
  - Сравнение меток с музыкальной структурой: припев часто представлен наиболее повторяющимися сегментами

### Для Librosa-бэкенда:
- Наиболее полезные параметры для анализа:
  - `similarity_scores`: Показывает, какие сегменты похожи на другие части трека (повторяемость)
  - `hpss_harmonic_energy`: Помогает определить сегменты с высокой мелодической активностью
  - `tempogram_stability`: Показывает ритмическую стабильность сегментов (припевы часто имеют более стабильный ритм)
  - `total_segment_scores`: Комбинированная метрика для оценки "припевности" сегмента
  - `threshold`: Порог для классификации, который можно анализировать для настройки чувствительности

### Для гибридного подхода:
- Сравнение результатов обоих бэкендов:
  - `confirmed_by`: Показывает, какой бэкенд подтвердил сегмент как припев
  - Возможность анализа расхождений между методами для улучшения точности

## Возможности расширения для определения других типов сегментов

### Для `_detect_librosa_with_info`:
- Для определения дополнительных типов сегментов (проигрыш, тишина, бридж, припев 2 и т.д.) можно:
  - Использовать более сложную классификацию на основе комбинации признаков:
    - Низкая `hpss_harmonic_energy` может указывать на тишину или инструментальный проигрыш
    - Низкая `tempogram_stability` может указывать на бридж или переходные сегменты
    - Средние значения `similarity_scores` могут указывать на куплеты
  - Применить кластеризацию на основе всех признаков (similarity, hpss, tempo) для автоматического определения типов сегментов
  - Использовать машинное обучение для классификации сегментов на основе их признаков
  - Добавить анализ энергии сигнала для определения тишины/проигрышей
  - Ввести дополнительные признаки, такие как спектральная сложность, темповые изменения и т.д.

### Для `_detect_msaf_with_info`:
- Для определения дополнительных типов сегментов можно:
  - Использовать все уникальные метки, а не только наиболее повторяющуюся:
    - Каждая уникальная метка может представлять отдельный тип сегмента (припев, куплет, бридж, проигрыш)
    - Повторяющиеся метки указывают на припев, одиночные - на куплеты или бриджи
    - Метки с низкой энергией (анализируемой дополнительно) могут быть проигрышами или тишиной
  - Добавить анализ акустических признаков для каждой метки (энергия, темп, спектральные признаки)
  - Использовать дополнительные алгоритмы MSAF для более детальной сегментации
  - Применить пороговую фильтрацию по количеству и длительности меток для классификации типов сегментов

### Общие рекомендации для расширения:
- Модифицировать класс `SegmentInfo` для поддержки дополнительных типов сегментов:
  - Добавить поле `segment_subtype` с возможными значениями: "verse", "chorus", "bridge", "intro", "outro", "instrumental", "silence", "pre_chorus", "post_chorus" и т.д.
- Создать правила классификации на основе комбинации признаков:
  - Высокая повторяемость + высокая энергия = chorus
  - Низкая повторяемость + средняя энергия = verse
  - Низкая энергия + низкая активность = silence
  - Низкая энергия + инструментальная активность = instrumental
  - Средняя повторяемость + высокая ритмическая стабильность = pre_chorus
