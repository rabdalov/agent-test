## Задача

Исследовать и выбрать оптимальный набор признаков на основе `librosa` для более точного определения припевов, заменив или дополнив текущий подход через `msaf`.

---

## Текущий подход (проблемы)

- [`ChorusDetector`](app/chorus_detector.py:14) использует `msaf.process()` с алгоритмами `sf` (границы) и `scluster` (метки).
- Критерий выбора припева — наиболее часто повторяющаяся метка сегмента.
- Нет учёта музыкальных характеристик: энергии, гармоники, ритма, тембра.
- Нет порогов по длительности и позиционных критериев.
- `msaf` требует monkey-patch для совместимости с `scipy 1.12+`.

---

## Варианты улучшения на основе `librosa` (исследовать и выбрать оптимальный набор)

### 1. Harmonic/Percussive Separation (`librosa.effects.hpss`)

- Разделить сигнал на гармоническую (мелодия, аккорды) и перкуссивную (ударные) составляющие.
- Припев, как правило, имеет более высокую энергию обеих составляющих.
- Использовать как дополнительный признак для подтверждения сегментов-кандидатов.

### 2. Chroma Features (`librosa.feature.chroma_cqt` / `chroma_stft`)

- Хроматограмма отражает распределение энергии по 12 полутонам.
- Припевы часто имеют схожие хроматические паттерны при каждом повторении.
- Self-similarity по хроме позволяет найти повторяющиеся структуры.

### 3. Self-Similarity Matrix (`librosa.segment.recurrence_matrix`)

- Матрица самоподобия показывает, насколько похожи друг на друга временны́е фреймы.
- Блоки высокого сходства вдоль диагонали соответствуют повторяющимся секциям (припевам).
- Можно использовать совместно с chroma или MFCC для построения матрицы.

### 4. Tempogram / Ритмическая стабильность (`librosa.feature.tempogram`)

- Темпограмма отражает ритмическую активность во времени.
- Припевы, как правило, имеют более стабильный и выраженный ритм по сравнению с куплетами и бриджами.
- Можно использовать как дополнительный фильтр для отсева нестабильных сегментов.

## План реализации

### Шаг 1. Добавить зависимость

```bash
uv add librosa
```

### Шаг 2. Реализовать вспомогательные методы в [`ChorusDetector`](app/chorus_detector.py:14)

| Метод | Описание |
|---|---|
| `_compute_chroma(audio_file)` | Вычисление хроматограммы |
| `_compute_self_similarity(chroma)` | Матрица самоподобия |
| `_compute_tempogram_stability(audio_file)` | Ритмическая стабильность по сегментам |
| `_hpss_energy(audio_file)` | Энергия гармоники и перкуссии по сегментам |

### Шаг 3. Обновить метод [`detect()`](app/chorus_detector.py:27)

Использовать комбинацию выбранных признаков для ранжирования сегментов-кандидатов.

### Шаг 4. Добавить параметры конфигурации в [`app/config.py`](app/config.py) и [`example.env`](example.env)

| Параметр | Значения | По умолчанию |
|---|---|---|
| `CHORUS_DETECTOR_BACKEND` | `msaf` \| `librosa` \| `hybrid` | `hybrid` |
| `CHORUS_MIN_DURATION_SEC` | число секунд | `15` |
| `CHORUS_MAX_DURATION_SEC` | число секунд | `60` |

### Шаг 5. Написать/обновить тест

Обновить `scripts/test_chorus_detector.py` для проверки на тестовых треках.

## Проверка

- На тестовых треках из `data_exapmles/` детектор корректно определяет временны́е отрезки припевов.
- При `CHORUS_DETECTOR_BACKEND=librosa` используется только `librosa`-подход.
- При `CHORUS_DETECTOR_BACKEND=msaf` используется только текущий `msaf`-подход.
- При `CHORUS_DETECTOR_BACKEND=hybrid` результаты `librosa` и `msaf` объединяются для повышения точности.
