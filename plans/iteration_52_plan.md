# Итерация 52: Документирование — User-Guide

## Цель
Создать руководство пользователя: как пользоваться ботом, команды, workflow, советы.

## Создаваемые документы

### `docs/user-guide/index.md`
Введение для пользователя:
- Что делает бот
- Как начать (/start)
- Минимальные требования к входным файлам
- Ограничения

### `docs/user-guide/workflow.md`
Полный рабочий процесс:

#### 1. Отправка файла или ссылки
- Поддерживаемые форматы: MP3, FLAC, MP4, MKV, AVI
- Поддерживаемые источники:
  - Прямой файл (загрузка в Telegram)
  - Ссылка на прямое скачивание
  - Яндекс.Музыка (music.yandex.ru)
  - YouTube (youtube.com, youtu.be)

#### 2. Выбор языка
- Почему важен язык
- Доступные варианты
- Влияние на транскрипцию

#### 3. Получение текста песни
- Автоматический поиск (Lyrica, Genius)
- Ручная загрузка текста
- Fallback на транскрипцию

#### 4. Ожидание обработки
- Сколько времени занимает (оценки)
- Стадии пайплайна
- Как проверить прогресс

#### 5. Получение результата
- Формат MP4 с аудиодорожками
- Ссылка на скачивание
- Отправка в Telegram

### `docs/user-guide/commands.md`
Полный список команд:

| Команда | Описание | Пример |
|---------|----------|--------|
| `/start` | Начало работы, приветствие | `/start` |
| `/help` | Справка по командам | `/help` |
| `/search` | Поиск трека | `/search` |
| `/continue` | Продолжить после ошибки | `/continue` |
| `/step_*` | Перезапуск с шага | `/step_align` |
| `/change` | Изменить тип сегмента | `/change 1-5` |

- Перенести и расширить из `docs/bot_commands.md`
- Добавить скриншоты (если возможно)

### `docs/user-guide/tips.md`
Советы и рекомендации:

#### Лучшие практики
- Оптимальная длительность трека (2-5 минут)
- Качество исходного файла
- Чёткость вокала для транскрипции

#### Ограничения
- Максимальный размер файла (ограничения Telegram)
- Языки с плохой поддержкой
- Сложные музыкальные жанры

#### Частые вопросы
- Почему текст не совпадает?
- Как исправить тайминг?
- Что делать при ошибке?
