# Итерация 25 — Исправление ошибок в уведомлениях пользователя по ходу pipeline

## Решаемая проблема

Сообщения, не являющиеся фиксированными шагами pipeline, остаются отдельными сообщениями, а не редактируются последующими шагами.

Примеры проблемных сообщений:
- `"Загружаю трек с Яндекс Музыки..."`
- `"Трек загружен с Яндекс Музыки!"`
- `"Не удалось автоматически найти текст песни"`
- `"Текст песни получен. Продолжаю обработку ..."`

Каждое из этих сообщений отправляется как новое сообщение в чат, что приводит к засорению диалога множеством промежуточных уведомлений.

## Решение

Изменить логику уведомлений по следующим правилам:

1. **Ответ на запрос пользователя** с новым треком или ссылкой формируется в виде `"Reply to"` — то есть бот отвечает непосредственно на сообщение пользователя.
2. **Все последующие уведомления** от бота редактируют первоначальный ответ на запрос пользователя, последовательно меняя статус по ходу pipeline.

## Ожидаемый результат

- На каждый трек от пользователя остаётся только **одно последнее сообщение** от бота.
- Все уведомления корректируют первое сообщение от бота для каждого трека.
- Диалог остаётся чистым и читаемым: пользователь видит актуальный статус обработки в одном месте.
