# Итерация 32 — Горячая перезагрузка конфигурации из `.env` без перезапуска бота

## Задача

При изменении файла `.env` новые значения параметров должны автоматически применяться в работающем боте без его перезапуска.

## Текущее состояние (проблема)

- Функция [`load_settings()`](app/config.py:224) вызывается один раз при старте приложения в [`app/main.py`](app/main.py).
- Объект `Settings` создаётся единожды и передаётся во все компоненты (`BotApp`, `KaraokeHandlers`, `KaraokePipeline` и т.д.).
- Изменения в `.env` не применяются до следующего перезапуска бота.

## Требования к реализации

1. Мониторинг файла `.env` на изменения (по `mtime`) — без сторонних библиотек (`watchdog` и т.п.), только стандартная библиотека Python.
2. При обнаружении изменения — перечитать `.env`, пересоздать объект `Settings` и обновить его во всех компонентах.
3. Перезагрузка не должна прерывать текущие активные пайплайны — новые значения применяются только к новым запросам.
4. Параметры, которые нельзя изменить без перезапуска (например, `TELEGRAM_BOT_TOKEN`), логируются с предупреждением, но не применяются.
5. Успешная перезагрузка логируется на уровне `INFO` с перечислением изменившихся параметров (без вывода секретных значений).

## Архитектурное решение

### Новый класс `ConfigWatcher` в [`app/config_watcher.py`](app/config_watcher.py)

- Хранит путь к `.env`, последний `mtime` и текущий объект `Settings`.
- Метод `check_and_reload()` — сравнивает `mtime`, при изменении вызывает [`load_settings()`](app/config.py:224) и обновляет `Settings`.
- Метод `get_settings()` — возвращает актуальный объект `Settings`.

### Фоновая задача в [`app/bot_app.py`](app/bot_app.py)

- Запустить фоновую `asyncio`-задачу через `asyncio.create_task`, которая каждые N секунд (настраивается через `ENV_RELOAD_INTERVAL_SEC`, по умолчанию `30`) вызывает `ConfigWatcher.check_and_reload()`.

### Обновление компонентов

- Все компоненты ([`KaraokeHandlers`](app/handlers_karaoke.py), [`KaraokePipeline`](app/pipeline.py)) получают `Settings` через `ConfigWatcher.get_settings()` при каждом новом запросе, а не хранят ссылку на объект.

## Параметры конфигурации

| Параметр | Описание | Значение по умолчанию |
|---|---|---|
| `ENV_RELOAD_INTERVAL_SEC` | Интервал проверки изменений `.env` в секундах | `30` |
| `ENV_RELOAD_ENABLED` | Включить/выключить горячую перезагрузку | `true` |

## Параметры, не подлежащие горячей перезагрузке (требуют перезапуска)

| Параметр | Причина |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен бота используется при инициализации `Bot` |
| `LOG_LEVEL` | Уровень логирования настраивается один раз при старте |

## Формат лог-записей

```
INFO app.config_watcher: .env изменён, перезагружаю конфигурацию...
INFO app.config_watcher: Конфигурация обновлена. Изменились параметры: send_video_to_user, correct_transcript_enabled
WARNING app.config_watcher: Параметр TELEGRAM_BOT_TOKEN изменён, но требует перезапуска бота для применения.
```

## Проверка

1. **Горячая перезагрузка рабочего параметра:** изменить значение `SEND_VIDEO_TO_USER` в `.env` во время работы бота — через не более чем `ENV_RELOAD_INTERVAL_SEC` секунд новое значение применяется к следующему запросу без перезапуска.
2. **Защищённый параметр:** изменить `TELEGRAM_BOT_TOKEN` — бот логирует предупреждение, но продолжает работать со старым токеном.
3. **Некорректный `.env`:** синтаксическая ошибка в файле — бот логирует ошибку и продолжает работать с предыдущей конфигурацией.
