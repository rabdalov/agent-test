# ADR-001: Выбор менеджера зависимостей (UV vs Conda)

**Статус:** ✅ Принято  
**Дата:** 2026-03-19  
**Итерация:** 48

---

## Контекст

Проект караоке-бота использует ML-зависимости (demucs, librosa, msaf, torch). Возник вопрос о целесообразности перехода с UV на Conda/Mamba для управления зависимостями, особенно с учётом планируемого добавления Montreal Forced Aligner (MFA), который распространяется только через conda-forge.

## Рассмотренные варианты

| Вариант | Описание |
|---------|----------|
| **A. Только UV** | Единый менеджер зависимостей для всего проекта |
| **B. Только Conda** | Полный переход на Conda/Mamba |
| **C. Гибридный** | UV для основного проекта + Conda для MFA |

## Решение

**Использовать только UV, отказаться от MFA.**

### Обоснование

Приоритет проекта — **максимальная стабильность инфраструктуры**. Интеграция MFA требует:
- Введения conda-окружения в дополнение к UV
- Усложнения Docker-инфраструктуры (multi-stage builds)
- Потери воспроизводимости и скорости сборки

Текущий `AlignmentService` с LRC-импортом и sequence alignment достаточен для задач проекта.

## Сравнение по критериям

| Критерий | UV | Conda | Гибрид |
|----------|-----|-------|--------|
| Скорость resolve | ⭐⭐⭐ | ⭐ | ⭐⭐⭐ |
| Размер образа | ⭐⭐⭐ | ⭐⭐ | ⭐⭐ |
| Воспроизводимость | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ |
| Сложность инфраструктуры | ⭐⭐⭐ | ⭐⭐⭐ | ⭐ |
| MFA поддержка | ❌ | ⭐⭐⭐ | ⭐⭐⭐ |

### Почему не гибридный подход?

| Фактор | Влияние |
|--------|---------|
| Сложность | Два менеджера зависимостей, multi-stage Docker |
| Поддержка | Сложнее отлаживать, больше точек отказа |
| Выгода | Незначительное улучшение alignment |
| Воспроизводимость | Потеря детерминированности |

## Последствия

### Положительные
- Единый источник зависимостей: `pyproject.toml` + `uv.lock`
- Быстрая сборка и воспроизводимость
- Компактный Docker-образ (~200-300 MB)
- Простота инфраструктуры

### Отрицательные
- Отказ от потенциального улучшения alignment через MFA
- Ручное управление CUDA при необходимости GPU

## Реализация

```dockerfile
# Dockerfile — единственный базовый образ
FROM python:3.12-slim

# Установка UV
COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /bin/

# Зависимости
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-dev

# Приложение
COPY app/ ./app/
CMD ["uv", "run", "python", "-m", "app.main"]
```
