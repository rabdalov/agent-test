# План реализации сервиса выравнивания таймкодов

## Задача
Разработать сервис, который выравнивает привязку таймкодов файла с результатами транскрипции (результат шага `TRANSCRIBE`) и полного текста песни (результат шага `GET_LYRICS`). Учесть, что полный текст также может содержать тайм-коды в формате:
```
[00:10.86] Ya leel
[00:18.21] 
[00:35.79] Hadaee mada tawila
```

## Цель
Создать надежный механизм сопоставления временных меток из транскрибации с точным текстом песни, позволяющий генерировать синхронизированные субтитры для караоке-приложения.

## Подходы реализации

### 1. Выравнивание последовательностей слов (Sequence Alignment)

#### Как работает:
- Строится матрица соответствия между словами транскрибации и точного текста
- Находится оптимальное глобальное выравнивание, минимизирующее количество несоответствий, вставок и удалений
- Временные метки из транскрибации переносятся на выровненные слова точного текста
- При отсутствии слова в транскрибации (пропуск), интерполируется время между соседними словами

#### Техническая реализация:
- Использовать алгоритмы динамического программирования (модификация алгоритма Нидлмана–Вунша)
- Применить библиотеки: `difflib`, `needleman-wunsch`
- Реализовать функцию сопоставления слов с учетом возможных различий в написании и орфографии

#### Преимущества:
- Не требует аудиофайл для работы
- Быстрое выполнение
- Хорошо работает при наличии качественной транскрипции

#### Недостатки:
- Меньшая точность по сравнению с акустическими методами
- Может давать ошибки при значительных расхождениях в текстах

### 2. Forced alignment с использованием готовых инструментов

#### Как работает:
- Установка одного из специализированных инструментов (Montreal Forced Aligner, Gentle, Penn Phonetics Lab Forced Aligner)
- Подготовка аудио и текстового файла (в формате, где текст разбит на строки или слова)
- Инструмент использует акустические модели и словарь произношений для нахождения точных границ слов в аудио

#### Техническая реализация:
- Интеграция с MFA (Montreal Forced Aligner) через Python API или командную строку
- Подготовка аудиофайла и текста в нужном формате
- Обработка результатов и преобразование в формат, совместимый с системой

#### Преимущества:
- Высокая точность выравнивания
- Использует акустическую информацию для более точного определения границ слов
- Лучше справляется с фонетическими вариациями

#### Недостатки:
- Требует наличия аудиофайла
- Более медленная обработка
- Сложнее в установке и настройке

```python
            AlignedLyricsResult: Выровненные таймкоды для текста песни

```

### Стратегии выравнивания
```python
class AlignmentStrategy(ABC):
    @abstractmethod
    def align(self, transcription_words: List[WordWithTimestamp], lyrics_text: str) -> List[WordWithTimestamp]:
        pass

class SequenceAlignmentStrategy(AlignmentStrategy):
    """Выравнивание последовательностей слов"""
    pass

class ForcedAlignmentStrategy(AlignmentStrategy):
    """Forced alignment с использованием аудио"""
    pass
```

## План реализации

### Этап 1: Подготовка инфраструктуры
1. Создать классы моделей данных для хранения таймкодов
2. Реализовать парсер таймкодов из различных форматов
3. Подготовить тестовые данные для проверки алгоритмов

### Этап 2: Реализация Sequence Alignment
1. Реализовать алгоритм Нидлмана-Вунша для выравнивания последовательностей слов
2. Добавить обработку различных форматов таймкодов
3. Реализовать интерполяцию времени для пропущенных слов
4. Протестировать на различных примерах

### Этап 3: Интеграция Forced Alignment
1. Исследовать возможности интеграции MFA, который развернут в отдельном докер контейнера и доступен по ip: 192.168.0.206
2. Реализовать wrapper для вызова MFA
3. Обеспечить совместимость форматов данных

### Этап 4: Комбинирование подходов
1. Реализовать логику выбора стратегии в зависимости от доступных данных
2. Добавить валидацию результатов
3. Реализовать финальный сервис с обработкой ошибок

## Форматы данных

### Входные данные
- Транскрипция: `List[{'word': str, 'start_time': float, 'end_time': float}]`
- Текст песни: 
    a. строки с возможными таймкодами в формате `[MM:SS.mm] line of words`
    b. строки без таймкодов `line of words`
### Выходные данные
- Выровненный текст по словам: `List[{'word': str, 'start_time': float, 'end_time': float}]`
- Выровненный текст по строкам из текста песни: `List[{'line of words': str, 'start_time': float, 'end_time': float}]`

## Обработка особых случаев
1. Отсутствие таймкодов в обоих источниках
2. Различия в тексте (орфографические, смысловые)
3. Неполные транскрипции
4. Пустые или нулевые временные метки

## Критерии качества
Точность сопоставления слов и таймкодов

## Интеграция в общий pipeline

### Место в пайплайне
Согласно документу `docs/vision.md`, шаг выравнивания таймкодов (`ALIGN`) является частью основного пайплайна обработки:
`DOWNLOAD → SEPARATE → TRANSCRIBE → GET_LYRICS → ALIGN → GENERATE_ASS → RENDER_VIDEO`

Шаг `ALIGN` выполняется после:
- `TRANSCRIBE`: получения результата транскрибации с таймкодами (файл `transcribe_json_file`)
- `GET_LYRICS`: получения полного текста песни (файл `source_lyrics_file`)

Результат шага `ALIGN` сохраняется в файле `aligned_lyrics_file` и используется на следующем шаге `GENERATE_ASS` для создания файла субтитров в формате ASS.

### Интеграция с PipelineState
В модели `PipelineState` (описанной в `docs/vision.md`) добавляется поле:
- `aligned_lyrics_file` — путь к JSON с выровненным текстом и таймкодами (заполняется на шаге `ALIGN`)

### Взаимодействие с другими компонентами
- `KaraokePipeline` вызывает `AlignmentService` на шаге `ALIGN`, передавая ему пути к файлам транскрибации и текста песни
- `AlignmentService` использует результаты предыдущих шагов и обновляет состояние пайплайна
- Результаты выравнивания передаются на следующий шаг `GENERATE_ASS` для создания файла субтитров

### Обработка ошибок и продолжение
- При ошибке на шаге `ALIGN` пайплайн сохраняет информацию об ошибке в `PipelineState`
- Возможность продолжения обработки с шага `ALIGN`, если промежуточные файлы доступны
- В случае неудачи на шаге `ALIGN` пользователю предоставляется соответствующее сообщение об ошибке

### Форматы файлов в контексте пайплайна
- Вход: JSON-файл с результатами транскрибации (из шага `TRANSCRIBE`) и TXT-файл с текстом песни (из шага `GET_LYRICS`)
- Выход: JSON-файл с выровненными таймкодами, готовый для использования на шаге `GENERATE_ASS`

### Взаимодействие с Audio файлом
- В случае использования forced alignment, сервис может получить доступ к аудиофайлу через `PipelineState.vocal_file` (результат шага `SEPARATE`)
- При невозможности использовать аудио, применяется резервный метод sequence alignment
